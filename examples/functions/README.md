# Functions Examples

This folder contains the actual example apps used throughout the docs.

## Quick guide

- Use [node/](node/README.md), [python/](python/README.md), [php/](php/README.md), [rust/](rust/README.md), [go/](go/README.md), and [lua/](lua/README.md) to find runtime-specific demos
- Use [assets-static-first](assets-static-first/README.md), [assets-spa-fallback](assets-spa-fallback/README.md), and [assets-worker-first](assets-worker-first/README.md) for public asset and SPA behavior
- Use [next-style](next-style/README.md), [rest-api-methods](rest-api-methods/README.md), [versioned-api](versioned-api/README.md), and [platform-equivalents](platform-equivalents/README.md) for higher-level app patterns
- Use [node/auto-infer-node-multi-deps](node/auto-infer-node-multi-deps/README.md) and [python/auto-infer-python-multi-deps](python/auto-infer-python-multi-deps/README.md) if you want to compare explicit manifests with optional inference backends
- Use [polyglot-tutorial](polyglot-tutorial/README.md) for a guided learning path

## What to expect

- Most examples are small and runnable
- Some examples are focused on one concept only
- Some examples show a larger end-to-end flow and may have more setup
- The inference demos are intentionally simple and are best read as bootstrap examples, not as a recommendation to skip manifests forever
