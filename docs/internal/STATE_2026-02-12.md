## Key Gotchas / Root Causes

### 1) Running `docker compose` from the wrong directory

If the prompt is something like `endpoint % ...`, commands like `docker compose exec ...` will fail because there is no compose file in that directory.

Correct:

```bash
cd /Users/misaelzapata/Downloads/fastfn
docker compose up -d --build
```

### 2) `$TELEGRAM_BOT_TOKEN` expanded on the host (empty) instead of inside container

If the command is inside double-quotes on the host, `$TELEGRAM_BOT_TOKEN` expands before `sh -lc` runs in the container.

Correct (escape `$` or use single-quotes around the whole command):

```bash
cd /Users/misaelzapata/Downloads/fastfn && \
docker compose exec -T openresty sh -lc \
'wget -qO- "https://api.telegram.org/bot\$TELEGRAM_BOT_TOKEN/deleteWebhook?drop_pending_updates=true"'
```

### 3) Host loopback calls can fail in sandboxed terminals

In this Codex environment, host `curl http://127.0.0.1:8080/...` fails with:

- `Immediate connect fail ... Operation not permitted`

Workaround (for verification/debug): run the HTTP request from inside the container:

```bash
docker compose exec -T openresty sh -lc "curl -sS http://127.0.0.1:8080/_fn/health"
```
