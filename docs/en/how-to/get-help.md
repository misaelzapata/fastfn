# Get Help

> Verified status as of **March 13, 2026**.

## Quick View

- Complexity: Beginner
- Typical time: 5 minutes
- Outcome: faster issue triage and support handoff

## Support Channels

1. GitHub issues for bugs and feature requests
2. Pull request discussions for code-specific review
3. Public documentation and troubleshooting guides

If you need a faster starting point, use:

- [Run and test](./run-and-test.md)
- [Debugging and troubleshooting](./debugging-and-troubleshooting.md)

## Triage Checklist

Before opening an issue:

- include exact command and error output
- include runtime mode (`dev` vs `dev --native`)
- include platform and version details
- include minimal reproducible function path

## Minimal bug report template

```text
FastFN version:
Mode:
OS:
Command:
Expected:
Actual:
Repro steps:
```
