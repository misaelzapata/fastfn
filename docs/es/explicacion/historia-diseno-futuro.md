# Historia, Diseno y Futuro

> Estado verificado al **13 de marzo de 2026**.

## Vista rapida

- Complejidad: Conceptual
- Tiempo tipico: 8-12 minutos
- Resultado: contexto de decisiones de diseno y direccion de roadmap

## Rationale de diseno

- file-system routing para ownership simple
- runtimes poliglotas con contrato HTTP uniforme
- visibilidad OpenAPI para desarrollo local y produccion

## Tradeoffs

- simplicidad operativa alta vs menos abstracciones de framework
- composicion explicita en lugar de DI por decoradores

## Direccion futura

- mejoras de despliegue productivo
- mayor cobertura y paridad de docs
- observabilidad y guardrails de runtime
