Getting Started
From zero to API in 60 seconds.
Install the CLI, create a project, write a function, run it. That's the whole workflow.

Install the CLI Step 1

A single binary. No runtime dependencies. Pick your method.

Terminal
# Homebrew (macOS / Linux)
$ brew install fastfn

# or npm (any platform)
$ npm install -g fastfn

# or download the binary directly
$ curl -sSL https://fastfn.dev/install.sh | sh

Create a project Step 2

One command scaffolds the project with an example function ready to go.

Terminal
$ fastfn init my-api
$ cd my-api

# project structure:
my-api/
  functions/
    hello/
      get.py  ← your first function
  fastfn.toml

Write a function Step 3

Just a file with a main(req) function. No decorators, no imports, no config.

functions/hello/get.py
1def main(req):
2 return {"message": "Hello, world!"}

Run it Step 4

The dev server starts with hot reload. Your function is immediately live.

Terminal
$ fastfn dev .

FastFn v0.1.0
Gateway ready at http://localhost:8080
Console ready at http://localhost:8080/console
Watching for changes...

$ curl http://localhost:8080/hello
{"message": "Hello, world!"}
That's it. GET /hello is live. No YAML, no deploy, no waiting.
The idea
Think Next.js App Router — for any backend language.
Your file structure defines your API. The file name sets the HTTP method. The folder path sets the route.
01

Files = Routes

Put a file in functions/, and it becomes an endpoint. Folders become path segments.

# file:
functions/users/get.js

# becomes:
GET /users
02

Dynamic Params

Use [brackets] for dynamic segments. Just like Next.js, but server-side.

# file:
functions/users/[id]/get.py

# matches:
GET /users/42 → req.params["id"] = "42"
03

Any Language

The file extension picks the runtime. Mix Python, Node, PHP, Lua, Rust, Go in one project.

functions/ai/predict/post.py
functions/auth/verify/post.rs
functions/webhook/post.js
functions/report/get.php
Polyglot
Same interface,
every language.

Every runtime gets the same Request object with params, headers, body, and query. Return a dict/object and FastFn serializes it to JSON.

  • Python for AI & data pipelines
  • Node.js for real-time & streaming
  • Rust for CPU-intensive workloads
  • PHP for rapid web endpoints
  • Go for concurrent services

Your folders are your API.

Features
Built for developers who ship fast.

Zero Config

No YAML. No Dockerfiles. No serverless.yml. Drop a file, write a handler — it's live.

OpenResty Gateway

Nginx + Lua gateway with < 2ms routing. Shared dict caching, epoll, high concurrency out of the box.

Hot Reload

fastfn dev watches your files and reloads instantly. No build step, no restarts, no waiting.

Warm Worker Pools

Runtimes stay alive with persistent workers via Unix domain sockets. Zero cold starts.

Auto OpenAPI

Your file structure generates Swagger documentation automatically. Browse it at /console.

Process Isolation

Each function runs in its own process. A crash in one never affects others. Clean isolation by default.
