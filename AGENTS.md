# AGENTS.md

Guidelines for coding agents working in this repository.

## Scope

- Make focused changes tied to the user request.
- Do not modify unrelated files.
- Do not add temporary artifacts, scratch files, or local-only outputs to the repo.

## Documentation Standard

- Do not create ad-hoc docs in random locations.
- User-facing docs go under `docs/en` and `docs/es` only when explicitly requested.
- MkDocs theme overrides live under `docs/overrides` (not `docs_overrides`).
- Agent process/coordination rules belong in this `AGENTS.md`.
- Benchmark raw outputs belong in `tests/stress/results` as data artifacts (`.json`, `.csv`, `.md` generated by scripts), not standalone narrative docs unless requested.

## Repository Layout

- Keep root layout canonical: `docs/`, `examples/`, `tests/` for main content.
- Place manual scripts under `tests/manual/`.
- Place local development sample fixtures under `tests/fixtures/local-dev-samples/`.
- Place generated UI/e2e artifacts under `tests/results/`.
- Do not reintroduce old top-level folders: `test/`, `test-integration/`, `test-results/`, `docs_overrides/`.

## Code Changes

- Prefer minimal, reversible changes.
- Avoid hardcoded machine-specific paths or assumptions.
- Keep native and embedded runtime behavior aligned when touching runtime/openresty code.
- If changing contracts (OpenAPI, runtime request/response, internal endpoints), add/update tests in the same change.

## Tests and Verification

- Run targeted tests for touched areas before finishing.
- For CLI/runtime changes, prefer:
  - `cd cli && go test ./...` (or scoped packages)
  - relevant integration scripts in `tests/integration/`
- If an expected tool is missing locally (for example Docker, coverage tooling), report it clearly.

## CI/CD Expectations

- Keep CI scripts reproducible and non-interactive.
- Avoid introducing steps that depend on local machine state.
- Release automation must fail fast when required secrets are missing.

## Repo Hygiene

- Keep the working tree clean of generated junk.
- Do not commit local caches, virtualenv internals, or build outputs.
- Do not remove or revert user changes that are unrelated to the requested task.
